Project goal

The project is intended to help caretakers distinguish cats that look similar, maintain a cleaner colony record, and identify which cats may already be known, missing, new, clipped, unclipped, or in need of follow-up.

What it should do

Identify cats

Compare uploaded photos against known colony members and return likely matches.

Track TNR status

Use ear-tip/clipped-ear signals and caretaker notes to help distinguish altered and unaltered cats.

Build a colony record

Store photos, names, markings, sightings, and health notes for each cat.

Handle lookalikes

Prioritize features like coat pattern, ear shape, facial markings, body markings, and tail pattern.

Support field use

Design around quick photo review and low-friction updates from caretakers.

Improve over time

Expand the training set with clipped and unclipped examples and known individual identities.

Model approach

The practical first version should avoid overclaiming. A good MVP is not a perfect cat recognizer; it is a ranked assistant that says: this cat may be one of these known cats, here are the visual reasons, and here is where a human caretaker should confirm.

  • Start with labeled folders of known cats and clipped/unclipped ear examples.
  • Use image embeddings to compare visual similarity across photos.
  • Add a separate ear-tip classifier for TNR status.
  • Keep human confirmation in the loop for final identity decisions.
  • Log uncertain predictions so the dataset improves over time.

Why it matters

Community cat management depends on knowing which cats are already part of a colony, which cats are new, which cats are altered, and which cats may need care. A lightweight identification tool can reduce duplicate records and make caretaker handoffs easier.
